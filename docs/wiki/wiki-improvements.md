# 위키 개선 아이디어

남은 작업 체크리스트. 사용자 입력, 판단, 초안 작성으로 묶었습니다.

## 사용자 입력이 필요한 항목

### 정량 지표 추가

`이미지 업로드`처럼 측정값이 붙으면 설득력이 생깁니다. 실측이 어렵다면 근사치라도 가능합니다.

- [ ] FSD 전환 전/후 빌드 시간, 파일 수, 순환 의존 감지 건수
- [ ] Barrel 제거 전/후 번들 사이즈, cold start 시간
- [ ] React Query staleTime/gcTime 설정 근거 (실측 또는 허용 기준)
- [ ] 글로벌 에러 핸들링 도입 전/후 try-catch 건수

### 정책 문서 실패 사례

각 문서에 "어떤 실패가 이 규칙을 만들었는가"를 한두 문단으로 추가. 기억나는 에피소드를 불릿으로 주시면 다듬습니다.

- [ ] 컴포넌트 패턴
- [ ] 레이아웃과 라우팅
- [ ] Modal 외부 클릭

## 판단이 필요한 항목

### ADR 승격 후보

규칙은 굳어 있지만 두 문서 다 umbrella 성격이라 단일 결정 단위가 아니어서 판단 필요. 승격 시 상태 Review → Accepted, Sidebar 카테고리 이동, `상황` → `배경`으로 섹션명 정렬이 따라옵니다.

- [ ] 인증 전략 검토 승격 여부 결정
- [ ] 다단계 폼 승격 여부 결정

### React Query 운영 후속 액션

문서 작업이 아니라 실제 구현. 진행 후 결과를 문서에 반영합니다.

- [ ] Query Key Factory 도입 PR (전/후 invalidate 실수 건수 비교)
- [ ] useSuspenseQuery 일괄 전환 (Suspense fallback 경험 사례 수집)

## 신규 문서 초안 후보

### 추가 주제

각 항목 초안은 가능하나 의도와 배경은 사용자 확인 필요.

- [ ] 번들 최적화와 코드 스플리팅 (`React.lazy` 의도와 측정 근거)
- [ ] 배포와 환경 (`devmkki.cloud`, Vercel, OCI VM, 환경별 분기)
- [ ] 로컬 개발 환경 구축 (HTTPS, CORS, 쿠키 설정)
- [ ] 테스트 전략 (단위/통합/E2E 채택 또는 미채택 사유)
- [ ] 접근성 (시맨틱 마크업, 키보드 포커스, 스크린 리더)
- [ ] 디자인 시스템과 Storybook 운영 (`mkki.github.io/status-front`)
- [ ] 성능 측정/모니터링 (Web Vitals, 에러 리포팅)
- [ ] 레이어 역전 해소 기록 (`shared/api` → `features/auth` 정리 후속편)

### 코드베이스에서 뽑을 트러블슈팅

git log와 현재 구조에 들어가 있지만 문서화되지 않은 소재. "사용자가 봤을 때 심각하게 어색했던" 케이스 위주로.

- [ ] Barrel 파일 제거 과정과 부작용
- [ ] FSD 마이그레이션 슬라이스와 의사 결정 흐름
- [ ] Vite 프록시 설정 제거 배경 (커밋 `chore: vite proxy 설정 제거`)
- [ ] 바텀 시트 dimmed 컬러 수정 배경 (커밋 `fix: 바텀 시트 dimmed 컬러 수정`)
- [ ] 반대로 적용됐던 회원 탈퇴 경고 문구 사건
