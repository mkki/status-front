# 위키 동기화 정책

`status-front` 프로젝트에서 기능 추가나 리팩토링이 끝나면 `/Users/user/Documents/personal/status-front.wiki/`의 관련 문서를 점검하고 업데이트 제안을 한 번 띄웁니다.

## Why

위키는 포트폴리오 공개 문서로 파일 경로, 라인 번호, 규칙 서술이 코드 변경에 따라 빠르게 낡습니다. 매번 수동으로 위키를 체크하는 비용을 줄이고 코드와 문서 간 정합성을 유지하기 위함입니다.

## When

`src/` 변경이 완료된 직후. 단순 버그 픽스, 주석 변경, 스타일 토큰 조정처럼 위키 본문 결론에 영향이 없는 변경은 생략합니다.

## 점검 대상 매핑

- 인증/세션: `HTTP-Client-설계.md`, `인증-전략-검토.md`, `시크릿-모드-쿠키-인증.md`
- WebView/네이티브: `WebView-연동.md`, `이미지-업로드.md`, `Architecture-&-Structure.md` (MESSAGE_TYPES 변경 시)
- 데이터/상태: `전역-상태-관리.md`, `다단계-폼.md`, `서버-로컬-데이터-동기화.md`
- UI/스타일: `컴포넌트-패턴.md`, `Modal-외부-클릭.md`, `SCSS-Module-스타일-제어.md`
- 인프라/패턴: `레이아웃과-라우팅.md`, `글로벌-에러-핸들링.md`, `React-Query-운영.md`

## 점검 항목

- 파일 경로 인용: 이동/이름 변경된 파일을 위키에서 grep해 깨진 참조 찾기
- 라인 번호 인용: 인용된 라인이 여전히 같은 의미인지 확인
- 규칙/결정 서술: 새 변경이 위키 결정과 모순되는지, 위키 규칙을 갱신할 만한지 판단
- 새 패턴/스토어/훅: 위키에 등록할 만한 새 추상이 추가됐는지 확인

## 처리 방식

- 자동 수정 금지. 영향 범위와 수정안을 짧게 보고하고 사용자 승인을 받은 뒤 반영
- 영향이 없으면 "위키 영향 없음" 한 줄로 종료
- 새 트러블슈팅이나 결정 문서가 필요해 보이면 위키의 `_Template.md` 기반 초안 작성을 제안

## 작업 시 함께 적용되는 규칙

- 새 문서는 위키 `_Template.md`의 섹션 순서와 상태 라벨 5종(Proposed/Review/Accepted/Resolved/Superseded)을 따릅니다
- 문체와 AI 티 회피 규칙은 사용자 메모리에 별도로 정의되어 있고 위키 작성 시 동일하게 적용됩니다
