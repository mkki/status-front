### #quest_config

| field | value |
| --- | --- |
| quest_num | 3 |
| select_attribute_num | 1 |
| output_theme_num | 4 |
| output_mainquest_num | 4 |
| output_subquest_num | 4 |
| select_subquest_num | 3 |

- quest_num: 유저가 한번에 진행할 수 있는 최대 **메인** 퀘스트 수
- select_attribute_num: [능력치 선택]화면에서 유저가 선택할 수 있는 **최대** 능력치 수
    - 유저는 select_attribute_num개를 선택할 수 있다.
- output_theme_num: [테마 선택]화면에서 출력되는 theme_id 개수
    - 출력은 theme_id의 theme_name (테마 이름 텍스트)을 출력함 (이하 메인퀘, 서브퀘 동일)
- output_mainquest_num: [메인 퀘스트 선택]화면에서 출력되는 mainquest_id 개수
- output_subquest_num: [서브 퀘스트 선택]화면에서 출력되는 mainquest_id 개수
- select_subquest_num: [서브 퀘스트 선택]화면에서 유저가 선택할 수 있는 서브 퀘스트 수
    - 유저는 1~select_subquest_num개를 선택할 수 있다.

## 1. 개요

퀘스트는 유저의 자기 계발을 유도하는 상태창 웹앱의 핵심 콘텐츠이다.

퀘스트는 메인 퀘스트와 서브 퀘스트로 구성되며, 모두 '퀘스트 만들기' Flow에서 유저가 설정할 수 있다.

즉, 퀘스트 = 메인 퀘스트 + 서브 퀘스트를 통합해서 지칭하는 용어이다.

- 메인 퀘스트
    - 목표와 같은 개념이다.
    - 퀘스트 설정 Flow에서 1개의 메인 퀘스트를 생성한다.
- 서브 퀘스트
    - 챌린지와 같은 개념이다.
    - 퀘스트 설정 Flow에서 설정한 1개의 메인 퀘스트에 대해 1~select_subquest_num개의 서브 퀘스트를 설정한다.

퀘스트 메뉴에서 '새로운 퀘스트' 버튼을 클릭해 퀘스트 설정 Flow에 진입한다.

## 2. 퀘스트 생성

## 3. 퀘스트 메뉴

- 진행 중인 퀘스트 인증

퀘스트 메뉴는 좌측에서 2번째에 위치한다.

- disable: 현재 진행 중인 퀘스트의 수가 quest_config 시트의 quest_num와 같거나 초과했을 때

#### 3.2.2 진행 중인 퀘스트 개수

#### 3.2.3 진행 중인 퀘스트 목록

진행 중인 퀘스트 중 수행 기한이 짧게 남은 것부터 상단에 출력한다.

남은 수행 기한이 동일한 퀘스트의 경우, 먼저 생성된 것(=오래된 순)을 상단에 출력한다.

출력 정보는 다음과 같다.

**1) 총 수행 기한**

Text: 기한_{0:s} (총 {1:d}주)

- 0: 설정한 목표 수행 기한일 출력
    - YYYY.MM.DD 형태로 출력
- 1: 설정한 목표 수행 기한이 총 몇 주인지 출력

**2) 메인 퀘스트 이름**

생성한 메인 퀘스트의 이름을 출력한다.

**3) 퀘스트 보상**

해당 메인 퀘스트의 보상을 출력한다.

link_type_1_id에 해당하는 능력치 보상을 좌측에 출력한다.

- 아이콘:
    - quest_mainquest 시트의 link_type_1_id에 맞는 능력치 아이콘을 출력한다.
    - quest_mainquest 시트의 link_type_2_id에 맞는 능력치 아이콘을 출력한다.
- 경험치:
    - Text: +{0:d}xp
        - 0:
            - quest_mainquest 시트의 link_type_1_xp 값을 출력한다.
            - quest_mainquest 시트의 link_type_2_xp 값을 출력한다.

**4) 메인 퀘스트 진행도**

Text: 진행도 {0:d}%

메인 퀘스트의 총 수행 기한 동안의 하위 서브 퀘스트의 수행 주기에 따른 총 횟수를 기준으로 현재까지의 진행도를 출력한다.

즉, '현재까지 수행한 서브 퀘스트 횟수'/'수행해야 하는 서브 퀘스트 총 횟수' * 100이다.

- 소수점은 모두 버린다.
- e.g.
    - 메인 퀘스트 총 수행 기간: 3주
    - 서브 퀘스트 A: 매일 → 총 21회(3*7)
    - 서브 퀘스트 B: 주 3회 → 총 9회 (3*3)
    - 서브 퀘스트 C: 월 2회 → 총 2회
    - 결과) 분모: 32

진행도에 맞춰 프로그레스 바를 우측 방향으로 출력한다.

최종적으로, 보상량은 다음과 같이 계산한다.

- 보상 1: link_type_1_id의 능력치를 (link_type_1_xp * 수행 횟수)만큼 지급
    - 수행 횟수: 다음의 표를 참고하여 xp에 가중치를 더한다.

    | 수행 횟수 | 입력 값(범위) | 가중치 |
    | --- | --- | --- |
    | time_second | 1이상 ~ 10이하 | 1 |
    |  | 11이상 ~ 30이하 | 1.1 |
    |  | 31이상 ~ 60이하 | 1.2 |
    | time_minute | 1이상 ~ 10이하 | 1 |
    |  | 11이상 ~ 30이하 | 1.1 |
    |  | 31이상 ~ 60이하 | 1.2 |
    |  | 61이상 ~ 120이하 | 1.3 |

    | time_hour | 1 ~ 3 | 1 |
    |  | 4 ~ 10 | 1.1 |
    |  | 11 ~ 12 | 1.2 |
    |  | 13 ~ 24 | 1.3 |
    | number_1 | 1~3 | 1 |
    |  | 4~10 | 1.2 |
    |  | 11~30 | 1.3 |
    |  | 31~99 | 1.4 |
    |  | 100~999 | 1.5 |
    | number_2 | 1~3 | 1 |
    |  | 4~10 | 1.2 |
    |  | 11~30 | 1.3 |

    | number_3 | 1~3 | 1 |

- (보상 1과 동일) 보상 2: link_type_2_id의 능력치를 (link_type_2_xp * 수행 횟수)만큼 지급

### 7.6 리롤 버튼

클릭 시 새롭게 output_subquest_num개의 subquest_id를 출력한다.

- 이때, 다음에 해당하는 서브 퀘스트는 리롤에서 제외 & 리롤 시 출력에서 제외한다.
    - 리롤에서 제외: 현재 selected되어 있는 subquest_id는 제외하고 나머지를 리롤한다.
    - 리롤 시 출력에서 제외: 이번에 출력했었던 subquest_id는 제외하고 나머지를 리롤하여 새로운 서브 퀘스트가 출력되도록 한다.
- 만약, 새롭게 출력 가능한 subquest_id의 개수가 output_subquest_num 미만일 때의 처리는 위 5.6 리롤 버튼의 처리와 동일하다.

새롭게 출력된 서브 퀘스트 목록에서는 아무 항목도 선택되어 있지 않은 Default 상태로 출력한다.

매우 빠르게 연속된 리롤을 방지하기 위해 다음 리롤 동작까지 1초의 쿨타임을 적용한다.

- 버튼 상태가 변경될 필요는 없음

### 7.7 다음 버튼

클릭 시 [퀘스트 만들기 - 수행 기간 설정 화면]으로 이동한다.

버튼의 active/disable 조건은 다음과 같다.

- active: 유저가 선택한 서브 퀘스트 항목이 최소 1개 ~ 최대 select_subquest_num개 만큼 selected 되어있는 경우 활성화한다.
- disable: 0개 또는 select_subquest_num개를 초과하는 개수가 selected 되어있는 경우 비활성화한다.

## 8. 퀘스트 만들기 - 수행 기간 설정 화면

### 8.1 UI 표시 방법

오늘을 기준으로 한 달력을 출력한다.

좌우의 달력 넘김 버튼 클릭 시 월별로 이동한다.

오늘을 포함하여 7일 째 되는 날을 활성화한 채로 출력한다.

- 최대 28일 째 되는 날까지 활성화한다. (즉, 4주째까지 활성화)

활성화된 날을 클릭하면 오늘부터 해당일까지의 기간을 모두 활성화한다.

#### 8.1.1 타이틀

Text: 퀘스트를 몇 주간 수행할까요?

#### 8.1.2 기한 표시

'오늘 날짜'~'선택한 날짜(총 n주)' 형태로 출력한다.

Text: {0:s} ~ {1:s} (총 {2:d}주)

- 0: 오늘 날짜(YYYY.MM.DD)
- 1: 선택한 날짜(YYYY.MM.DD)
- 2: 총 주차

날짜를 선택하지 않은 경우 '선택한 날짜(총 n주)'는 출력을 생략한다.

Text: {0:s} ~

### 8.2 수행 기한 설정

오늘을 포함하여 7일 단위로 수행 기한을 설정할 수 있다.

최소 7일(1주일) ~ 최대 28일(4주일) 중에서 선택한다.

### 8.3 완료 버튼

클릭 시 지금까지 진행한 퀘스트 만들기 Flow의 전체 정보를 가지고 퀘스트를 생성하며, [퀘스트 생성 완료] 화면으로 이동한다.

이때, 지금까지의 전체 정보가 저장되었는지 Validation 검사한다.

- 만약, 정보가 저장되지 않은 경우, 10.1 퀘스트 만들기 도중 에러 발생 시의 처리를 따른다.
    - 에러 페이지 출력

버튼이 enable/disable 되는 조건은 다음과 같다.

- enable: 기간을 선택했을 때
- disable
    - 기간을 선택하지 않았을 때
    - 1~4주 외의 기간을 선택했을 때

## 9. 퀘스트 만들기 - 퀘스트 생성 완료 화면

이번 생성 Flow에서 누적된 정보를 바탕으로 퀘스트를 생성한다.

### 9.1 퀘스트 NPC 이름

Text: From. [{0:s}]

- 0: quest_mainquest 시트의 mainquest_npc_name 출력

### 9.2 텍스트

Text: 생성된 퀘스트가 도착했어요!

### 9.3 캐릭터

캐릭터 이미지를 출력한다.

### 9.4 메인 퀘스트 정보

**1) 총 수행 기한**

Text: 기한_{0:s} (총 {1:d}주)

- 0: 설정한 목표 수행 기한일 출력
    - YYYY.MM.DD 형태로 출력
- 1: 설정한 목표 수행 기한이 총 몇 주인지 출력

**2) 메인 퀘스트 이름**

- 생성한 메인 퀘스트의 이름을 출력한다.

**3) 퀘스트 보상**

해당 메인 퀘스트의 보상을 출력한다.

link_type_1_id에 해당하는 능력치 보상을 좌측에 출력한다.

- 아이콘:
    - quest_mainquest 시트의 link_type_1_id에 맞는 능력치 아이콘을 출력한다.
    - quest_mainquest 시트의 link_type_2_id에 맞는 능력치 아이콘을 출력한다.
- 경험치:
    - Text: +{0:d}xp
        - 0:
            - quest_mainquest 시트의 link_type_1_xp 값을 출력한다.
            - quest_mainquest 시트의 link_type_2_xp 값을 출력한다.

### 9.5 서브 퀘스트 정보

선택한 서브 퀘스트의 반복 주기와 서브 퀘스트 이름을 출력한다.

subquest_id가 빠른 순서대로 상단부터 출력한다.

### 9.6 확인 버튼

클릭 시 [퀘스트] 화면으로 이동한다.

버튼이 disabled 상태가 되는 상황은 없다.

## 10. 공통

### 10.1 퀘스트 만들기 도중 에러 발생 시

퀘스트 만들기 Flow는 이전 화면의 입력 정보를 저장하고, 이전 스텝의 정보를 기반하거나, 최종 종합하여 퀘스트를 만드는 구조이다.

이에 따라, 이전 정보가 날라가는 등의 에러가 발생할 수 있다.

저장된 정보가 날라가는 경우 다음의 에러 페이지를 출력한다.

**1) 캐릭터**

에러 발생 시 캐릭터를 출력한다.

- 리소스는 Figma를 참고한다.

**2) 타이틀**

Text: 퀘스트 생성 실패

**3) 텍스트**

Text: 서버에 문제가 발생했어요.

Text: 새로운 퀘스트를 다시 만들어봐요.

**4) 이전으로 버튼**

클릭 시 [능력치 선택] 화면으로 이동한다.
