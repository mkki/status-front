# 퀘스트 인증 시스템

## 1. 퀘스트 상세 페이지

퀘스트 메뉴에서 진행 중인 퀘스트 항목을 클릭하면 해당 퀘스트의 상세 페이지로 이동한다.

### 1.1 뒤로가기 버튼

클릭 시 퀘스트 메뉴 홈으로 돌아간다.

### 1.2 타이틀

Text: 퀘스트 상세

### 1.3 퀘스트 포기 버튼

클릭 시 퀘스트 포기 팝업을 출력한다.

**1) 타이틀**

Text: 퀘스트를 포기할까요?

**2) 본문 Text**

Text: 삭제된 퀘스트는 되돌릴 수 없으며

Text: 중도 포기 시 보상을 받을 수 없습니다.

**3) 취소/포기하기 버튼**

**취소 버튼** 클릭 시 퀘스트를 삭제하지 않고, 팝업을 닫는다.

**포기하기 버튼** 클릭 시 해당 메인 퀘스트 및 서브 퀘스트를 삭제한다.

### 1.4 메인 퀘스트 정보

메인 퀘스트 정보를 출력한다.

출력 정보는 퀘스트 설정 시스템 - 9.4 메인 퀘스트 정보의 설명과 동일하다.

### 1.5 오늘의 퀘스트 정보

Text: 오늘의 퀘스트

오늘 인증하지 않은 서브 퀘스트를 subquest_id가 빠른 순서대로 위에서부터 출력한다.

- 오늘은 서버 시간 기준 00:00 ~ 23:59를 기준으로 한다.
- 남은 횟수를 모두 인증했더라도 '오늘 인증한 것이 아니면' 해당 영역에 출력한다.
    - e.g. 주 3회 조건(월~일) → 월, 화, 수에 인증을 했어도 목요일에 또 출력한다.
    - ※ 기획 의도: 더 하고 싶으면 할 수 있도록 열어둔다.
    - **(중요)** 계속해서 인증과 인증에 따른 서브퀘스트 보상을 획득할 수 있으나, 메인퀘스트 완료를 위한 총 횟수 카운트는 올리지 않는다.
        - 즉, 인증하더라도 서브퀘스트 보상만 받고, 메인퀘스트의 총 횟수에는 영향을 주지 않는다.

구성 정보는 다음과 같다.

**1) 서브 퀘스트 정보**

서브 퀘스트의 반복 주기와 서브 퀘스트 이름을 출력한다.

**2) 남은 횟수**

남은 수행 횟수를 다음과 같이 출력한다.

| 반복 주기 케이스 | Text |
| --- | --- |
| 매일 | *출력하지 않습니다.* |
| 주 n회 | (금주 {0:d}회 남음) |
| 월 n회 | (이번달 {0:d}회 남음) |

- 0: 해당 서브 퀘스트의 반복 주기에 따른 남은 횟수
    - 주 n회의 경우, 시작일 ~ 시작일 기준 6일째 되는 날을 기준으로 한 주를 카운트한다.
        - e.g. 수요일 시작: 수요일 ~ 다음주 화요일

**3) 필수 여부**

서브 퀘스트의 반복 주기를 기준으로 날짜를 계산해 필수 여부를 출력한다.

Text: 필수

| 반복 주기 케이스 | 조건 |
| --- | --- |
| 매일 | 오늘 안한 경우 출력 |
| 주 n회 | 시작일 기준 6일째 되는 날 - 남은 수행 횟수 ≤ 0인 날짜일 때 출력 |
| 월 n회 | 메인 퀘스트의 남은 기간 - 남은 수행 횟수 ≤ 0인 날짜일 때 출력 |

**4) 인증하기 버튼**

버튼 클릭 시 해당 서브 퀘스트의 인증 화면을 출력한다.

자세한 내용은 2. 인증 절차에서 설명한다.

### 1.6 오늘 인증한 퀘스트 정보

헤더 Text: 오늘 완료한 퀘스트

오늘 인증 완료한 퀘스트를 출력한다.

- 최근에 인증한 것이 상단에 출력한다.
    - 즉, 20시에 인증한 것이 12시에 인증한 것보다 상단에 위치한다.
    - 최초 인증을 기준으로 한다.
    - 최초 인증 이후 수정하더라도 순서가 변경되지 않는다.

영역 안에 출력하는 정보는 아래 1.7 인증 기록에서 설명한다.

### 1.7 인증 기록

Text: 인증 기록

인증 기록 항목은 인증 날짜가 최신인 것이 상단에 출력한다.

- 최초 인증 날짜를 기준으로 한다.

각 항목에는 최초 인증 날짜를 출력한다.

- YYYY.MM.DD 형태로 출력한다.
- 최초 인증 날짜 이후 수정하더라도 날짜는 수정되지 않는다.

#### 1.7.1 출력 정보

항목 클릭 시 아래 방향으로 펼친다.

- 항목 재클릭 시 접는다.

펼쳐진 영역에는 다음의 정보를 출력한다.

- 인증한 서브 퀘스트 중 subquest_id가 빠른 순서대로 상단부터 출력한다.

**1) 서브 퀘스트 정보**

서브 퀘스트의 반복 주기와 서브 퀘스트 이름을 출력한다.

**2) 편집 버튼**

클릭 시 해당 서브 퀘스트의 인증 내용을 편집할 수 있다.

서브 퀘스트의 인증 내용은 아래 2. 인증하기 절차에서 설명하는 내용을 따르며, 인증으로 저장된 데이터를 출력한다.

- 입력 및 버튼 클릭 시 동작은 2. 인증 절차와 동일하다.
    - 서브퀘스트 완료 보상은 제공되지 않는다.
- 변경된 정보로 오버라이드한다.
    - 단, 최초 인증 시점의 인증 여부, 날짜, 시간은 보관되어야 한다.

**3) 인증 정보**

아래 2. 인증 절차에서 설명하는 저장된 인증 정보를 출력한다.

## 2. 인증 절차

1.5 오늘의 퀘스트 정보에서 '인증하기' 버튼 클릭 시 인증하기 모달을 출력한다.

타이틀 Text: 퀘스트 인증하기

헤더 Text: [{0:s}]의 인증

- 0: 현재 선택한 서브퀘스트 이름

### 2.1 인증하기 타입

각 서브퀘스트는 quest_subquest 시트의 'confirm_type'의 enum에 따라 인증하기 버튼 클릭 시 출력하는 타입이 다르다.

현재(MS 1) 사용하는 타입은 'mood_log'타입만 사용한다.

#### 2.1.1 mood_log 타입

mood_log 타입은 기본적인 인증 타입으로, 간단한 인증을 요구하는 서브퀘스트일 때 사용한다.

다음으로 구성한다.

- 수행 난이도 - 필수
- 메모 - 선택

#### (MS 1 미구현) 2.1.2 photo_log 타입

※ MS 1 때는 photo_log 타입을 구현하지 않는다.

photo_log 타입은 여행, 운동 등 사진을 업로드하는 것이 인증의 주요한 방법으로 판단되는 서브퀘스트일 때 사용한다.

다음으로 구성한다.

- 수행 난이도 - 필수
- 사진 업로드 - 선택
- 메모 - 선택

### 2.2 수행 난이도

유저가 오늘 퀘스트를 하면서 느낀 체감 난이도를 선택하는 항목이다.

- ※ 기획 노트: **추후** 기록된 퀘스트의 수행 난이도를 기반으로 퀘스트 추천이나 최근 흐름을 분석하는데 사용될 수 있습니다.

헤더 Text: 수행 난이도 (필수)

좌측부터 아래 항목을 항상 고정된 위치에 출력한다.

- 난이도 하: "가뿐한 퀘스트"
- 난이도 중: "적당한 임무"
- 난이도 상: "전설적인 도전"

항목 클릭 시 selected 상태가 되며, 버튼 안에 색상 아이콘을 출력한다.

- 색상 아이콘은 Figma를 참고한다.

한번에 1개만 선택할 수 있다.

- 다른 항목 선택 시 기존 항목의 selected를 해제하고, 새롭게 선택한 항목을 selected로 바꾼다. (Radio 버튼 형식)

디폴트로 아무것도 선택되어있지 않은 상태로 출력한다.

### 2.3 메모

헤더 Text: 메모 (최대 300자)

Placeholder Text: 자유롭게 메모하세요.

- 아래가 모두 and조건으로 달성될 때 출력한다.
    - 아무 텍스트도 입력되지 않을 때 출력.
    - 인풋 박스가 활성화되지 않은 상태일 때.
- 또, 입력되었던 텍스트가 삭제되어 Null이 된 경우에 다시 출력한다.

유저가 자유롭게 텍스트를 입력할 수 있는 인풋 박스이다.

최대 300자까지 입력 가능하며, 최대 글자 수를 넘길 경우 더이상 텍스트 입력이 불가능하도록 한다.

### 2.4 완료(임무 보고) 버튼

버튼 Text: 이상, 퀘스트를 보고합니다.

해당 서브퀘스트의 'confirm_type' 별로 지정된 필수 항목의 입력이 완료된 경우 버튼을 활성화한다.

- 그렇지 않은 경우, 버튼은 비활성화되어야 한다.

클릭 시 서브퀘스트 보상 획득 팝업을 출력한다.

- **(중요)** 클릭 시점에 서브퀘스트 보상을 유저에게 지급한다.
    - 보상은 아래 3. 서브퀘스트 보상 획득 팝업에서 설명한다.
    - ※ 기획 의도: 버튼 클릭 시점이 해당 서브퀘스트를 서버에 인증하는 마지막 절차이기 때문에 이후 서브퀘스트 보상 획득 팝업 출력 중 에러나 강제 종료가 발생하더라도 보상이 정상 지급 되도록 하기 위함이다.

- **(중요)** 해당 서브퀘스트가 완료될 때 메인퀘스트가 완료되는 상황이라면, 메인퀘스트 보상도 서브퀘스트 보상 지급과 함께 지급한다.
    - 메인 퀘스트의 완료 조건은 다음과 같다.
        - 메인 퀘스트의 총 수행 기한 동안의 하위 서브 퀘스트의 수행 주기에 따른 총 횟수가 모두 인증된 경우
        - 즉, 수행해야 하는 서브 퀘스트 총 횟수가 충족된 시점에 메인퀘스트를 자동으로 완료 처리한다.

## 3. 서브퀘스트 보상 획득 팝업

**1) 보상 이미지**

서브퀘스트 보상 이미지를 출력한다.

- 이미지는 Figma를 참고한다.

**2) 텍스트**

Text: 서브퀘스트 완료 보상을 획득했어요!

**3) 보상 정보**

quest_subquest 시트에 입력된 해당 서브퀘스트의 reward_attribute_[n]_id에 대한 reward_attribute_[n]_xp 만큼의 경험치를 지급한다.

능력치 아이콘과 지급 경험치량 정보를 출력한다.

좌측에 reward_attribute_1_id의 정보를 출력한다.

- 아이콘: reward_attribute_[n]_id에 입력된 attribute_id의 아이콘을 출력한다.
- 경험치량: "+{0:d}xp" 형태로 출력한다.
    - 0: reward_attribute_[n]_xp

**4) 보상 받기 버튼**

Text: 보상 받기

클릭 시 **팝업을 닫고**, 이후에 다음의 두가지 케이스에 따라 처리한다.

(1) 메인 퀘스트가 완료되지 않은 경우

- 해당 서브 퀘스트가 속한 메인 퀘스트의 [퀘스트 상세] 화면으로 이동한다.
- ※ 참고: 이번에 완료한 서브 퀘스트는 '오늘 완료한 퀘스트' 영역에 출력되어야 합니다.

(2) 메인 퀘스트가 완료된 경우

- 보상 받기 버튼 클릭 시 메인퀘스트 보상 획득 팝업을 출력한다.

## 5. 완료 or 실패한 메인퀘스트 처리

### 5.1 완료한 메인퀘스트

완료한 메인퀘스트는 퀘스트 메뉴에서 정보를 출력하지 않는다.

추후 구현하는 히스토리 메뉴에 보관한다.

- 이번(MS 1)에 구현하지 않는다.
    - 이번에는 완료한 메인퀘스트는 퀘스트 메뉴에서 정보를 출력하지 않되, 서버 쪽에서 데이터를 삭제하지 않고 보관하는 처리까지 구현한다.
- (서버 확인) 얼마 동안 보관 가능한지 파악 필요

### 5.2 실패한 메인퀘스트

수행해야 하는 서브 퀘스트 총 횟수를 달성하지 못한 채 설정한 수행 기한이 지난 경우 해당 메인퀘스트는 실패 처리된다.

실패 시 다음의 처리를 한다.

- 메인퀘스트 보상은 지급하지 않는다.
- 마지막 수행 기한일 다음 날 00:00 부터 퀘스트 메뉴에서 출력하지 않는다.
    - 실패한 퀘스트도 추후 구현하는 히스토리 메뉴에 보관할 예정이기 때문에, 이번(MS 1)에는 서버 쪽에서 데이터를 삭제하지 않고 보관하는 처리까지 구현한다.
