# 히스토리 메뉴

## 1. 개요

히스토리 메뉴는 유저가 완료하여 저장된 퀘스트 정보를 출력하는 1Depth Menu이다.

- 완료한 퀘스트는 [퀘스트 인증 시스템 - 5. 완료 or 실패한 메인퀘스트 처리] 문서에 기재된 정의와 같다.
- 즉, 성공하거나 실패한 메인 퀘스트를 모두 출력하며, 메인 퀘스트와 서브 퀘스트의 수행 정보를 모두 출력한다.

히스토리 메뉴에 구성되는 정보는 다음과 같다.

- 1) 완료한 메인 퀘스트, 서브 퀘스트 기본 정보
- 2) 서브 퀘스트 인증 정보
- 3) 서브 퀘스트 인증 날짜
- 4) 메인 퀘스트 완료 날짜
- 5) 획득한 총 경험치
- 6) 수주 날짜

히스토리 메뉴에서 출력하는 정보는 수정이 불가능하다.

## 2. 메뉴 화면 UI

유저가 아직 메인 퀘스트를 하나도 완료하지 않은 경우 빈 화면을 출력한다.

완료한 퀘스트가 하나라도 있는 경우엔 누적 기록 + 완료한 메인 퀘스트 목록을 출력한다.

### 2.1 누적 기록

누적 기록은 완료한 퀘스트(성공 + 실패)의 데이터를 총합하여 계산한다.

**1) 완료한 총 메인 퀘스트 수**

- 성공+실패한 모든 메인 퀘스트의 개수를 출력한다.

**2) 총 서브 퀘스트 인증 횟수**

- 성공+실패한 모든 메인 퀘스트의 인증된 서브 퀘스트 개수를 출력한다.

**3) 평균 퀘스트 완료율**

- 성공+실패한 모든 메인 퀘스트의 완료율의 평균 값을 출력한다.
- 계산법: (모든 메인 퀘스트의 완료율 합산) / (모든 메인 퀘스트 개수)
- 출력 시 소수점은 버림 처리한다.

**4) 평균 퀘스트 수행 기간**

- 성공+실패한 모든 메인 퀘스트의 '퀘스트 시작일 ~ 완료일'의 평균 기간을 일 단위로 출력한다.
    - e.g. 999일
    - ※ 참고:
        - 퀘스트 시작일은 퀘스트 생성 시 설정 값을 가져오면 될 듯 합니다.
        - 퀘스트 완료일은 메인 퀘스트를 '최종 완료 or 실패한 날짜' 데이터를 가져와서 사용합니다.
- 계산법: (모든 메인 퀘스트의 수행 기간 합산) / (모든 메인 퀘스트 개수)

### 2.2 완료한 메인 퀘스트

완료한 메인퀘스트를 '완료일 기준 최근 순'으로 상단부터 하단 방향으로 출력한다.

메인 퀘스트 항목 클릭 시 퀘스트 상세(2.2.2 퀘스트 상세 정보 구성 목차) 화면을 출력한다.

#### 2.2.1 완료한 메인 퀘스트 정보 구성

[퀘스트 인증 시스템 - 1. 퀘스트 상세 페이지]의 내용과 유사하나, 다음의 정보로 다르게 출력한다.

**1) 성공/실패 여부**

- 해당 메인 퀘스트가 성공했는지 실패했는지에 따라 출력한다.

**2) 퀘스트 기간**

- 퀘스트 생성 시 설정한 퀘스트 시작일 ~ 마감일을 출력한다.

**3) 최종 진행도**

- 해당 메인 퀘스트의 최종 진행도를 출력한다.

#### 2.2.2 퀘스트 상세 정보 구성

[퀘스트 인증 시스템 - 1.7 인증 기록]의 정보 구성과 유사하나, 다음의 정보를 출력한다.

- 퀘스트 포기 버튼은 출력하지 않는다.

**1) 메인 퀘스트 정보**

- 위 2.2.1번 목차와 동일하게 메인 퀘스트 정보를 출력한다.

**2) 완료한 서브 퀘스트 정보**

- 수정이 불가능하도록 '수정 버튼'을 출력하지 않는다.

## 3. 정보 출력 방식

(임시 처리) 메뉴 입장 시 메뉴의 정보를 한번에 로딩하여 출력하는 방식으로 구현한다.

데모데이 이후 폴리싱 작업 때 섹션 로딩 방식으로 변경한다.
